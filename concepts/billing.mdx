GogoToken 采用预付费按量计费模式。充值余额后，每次 API 调用根据实际 Token 消耗从余额中扣除，没有月费或订阅。你只为真正使用的部分付费，随时充值、随时使用。

## 计费原理

Token 是 AI 模型处理文本的基本单位。你发送的输入文本（prompt）和模型返回的输出文本（completion）都会消耗 Token。不同模型的 Token 单价不同，旗舰模型（如 `claude-opus-4-8`）单价高于速度型模型（如 `claude-sonnet-4-6`）。

每次 API 调用完成后，系统会根据以下公式计算本次费用并从余额中实时扣减：

```
本次费用 = 输入 Token 数 × 输入单价 + 输出 Token 数 × 输出单价
```

每个模型的具体单价请以控制台价格页面为准，我们会在模型上线或调价时提前通知。

## 充值余额

在控制台的**余额充值**页面可以为账户充值。充值后余额实时到账，立即可用于 API 调用。

- **最低充值额度**：具体金额以控制台余额充值页面显示为准。
- **余额实时扣减**：每次 API 调用完成后立即扣除对应费用，无延迟。
- **用量明细**：在控制台的**用量记录**页面可查看每一次调用的 Token 消耗与扣费金额，数据精确到单次请求。

前往 [余额管理](/account/balance) 页面充值或查看余额详情。

## 余额与退款

账户余额对你完全透明，可在控制台随时查看当前余额及历史消费记录。

- **余额查询**：登录控制台，在首页概览或**账户 → 余额**页面可实时查看余额。
- **退款申请**：若你有未使用的余额需要退款，请联系我们的支持团队，我们将按原路退回。

<Note>
  账户余额**永久有效，不会过期**。你无需担心余额因闲置而被清零，随时可以继续使用。
</Note>

## 用量优化建议

合理使用以下技巧，可以在不影响效果的前提下显著降低 Token 消耗：

<Tip>
  设置合理的 `max_tokens` 参数来限制模型输出长度。如果你的场景不需要超长回复，将 `max_tokens` 控制在实际需求范围内可以有效减少输出 Token 消耗。
</Tip>

<Tip>
  对于分类、摘要、问答等**简单任务**，优先选择速度型模型（如 `claude-sonnet-4-6`）。速度型模型单价更低，在这类任务上的效果与旗舰模型相差无几。
</Tip>

<Tip>
  在**开发和测试阶段**使用简短的提示词，避免在调试过程中消耗大量 Token。待提示词逻辑验证完毕后，再切换为正式内容进行测试。
</Tip>
