Chat Completions 接口是 GogoToken 的核心接口，用于向 AI 模型发送消息并获取回复。支持单轮和多轮对话，以及流式（streaming）输出。该接口完全兼容 OpenAI `chat/completions` 规范，现有代码无需修改逻辑，仅需切换 Base URL 和 API Key 即可使用。

**端点**

```
POST https://gogotoken.org/v1/chat/completions
```

## 请求参数

<ParamField body="model" type="string" required>
  要使用的模型 ID。GogoToken 当前支持以下模型：`claude-opus-4-8`、`claude-opus-4-7`、`claude-sonnet-4-6`、`gpt-5-5`、`gpt-4o`。可通过 [GET /v1/models](/api/models) 接口获取最新可用模型列表。
</ParamField>

<ParamField body="messages" type="array" required>
  消息数组，代表本次对话的完整历史记录。每个消息对象包含以下字段：

  <Expandable title="messages 对象字段">
    <ParamField body="role" type="string" required>
      消息角色，可选值为 `"system"`、`"user"` 或 `"assistant"`。
      - `system`：系统提示词，用于设定模型的行为和角色
      - `user`：用户发送的消息
      - `assistant`：模型之前生成的回复（用于多轮对话）
    </ParamField>

    <ParamField body="content" type="string" required>
      消息的文本内容。
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="stream" type="boolean" default="false">
  是否启用流式输出（Server-Sent Events）。设置为 `true` 时，模型将逐步返回生成内容，而非等待完整回复后一次性返回。适合需要实时展示内容的场景。
</ParamField>

<ParamField body="max_tokens" type="integer">
  模型单次回复的最大输出 Token 数。超出此限制时，生成将被截断，`finish_reason` 将为 `"length"`。不同模型对此参数的上限不同。
</ParamField>

<ParamField body="temperature" type="number" default="1">
  控制输出的随机性与创造性，取值范围为 `0` 到 `2`。值越低，输出越确定、越保守；值越高，输出越随机、越有创意。建议在需要精确、可重复结果时使用较低值（如 `0.2`），在创意写作等场景中使用较高值（如 `1.2`）。
</ParamField>

<ParamField body="top_p" type="number">
  核采样（nucleus sampling）参数，取值范围为 `0` 到 `1`。模型在生成时只考虑累计概率达到 `top_p` 的 Token 集合。通常不建议同时调整 `temperature` 和 `top_p`。
</ParamField>

<ParamField body="stop" type="string | array">
  停止序列。当模型生成的内容中出现此字符串（或数组中的任一字符串）时，生成立即停止。最多可指定 4 个停止序列。
</ParamField>

<ParamField body="n" type="integer" default="1">
  为每条输入消息生成的候选回复数量。默认为 `1`。设置大于 `1` 的值时，`choices` 数组将包含多个结果，Token 消耗也将相应增加。
</ParamField>

## 请求示例

```json
{
  "model": "claude-opus-4-8",
  "messages": [
    {"role": "system", "content": "你是一个专业的技术顾问"},
    {"role": "user", "content": "解释一下 REST API 设计的最佳实践"}
  ],
  "max_tokens": 1000,
  "temperature": 0.7
}
```

使用 cURL 发送请求：

```bash
curl https://gogotoken.org/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-opus-4-8",
    "messages": [
      {"role": "system", "content": "你是一个专业的技术顾问"},
      {"role": "user", "content": "解释一下 REST API 设计的最佳实践"}
    ],
    "max_tokens": 1000,
    "temperature": 0.7
  }'
```

## 响应字段

<ResponseField name="id" type="string">
  本次请求的唯一标识符，格式如 `chatcmpl-abc123`。可用于日志记录和问题排查。
</ResponseField>

<ResponseField name="object" type="string">
  固定值 `"chat.completion"`，标识响应类型。
</ResponseField>

<ResponseField name="created" type="integer">
  请求创建时的 Unix 时间戳（秒）。
</ResponseField>

<ResponseField name="model" type="string">
  实际处理本次请求所使用的模型 ID。
</ResponseField>

<ResponseField name="choices" type="array">
  生成结果数组。默认包含 1 个元素，若请求中 `n > 1` 则包含多个。

  <Expandable title="choices 对象字段">
    <ResponseField name="choices[].index" type="integer">
      当前结果在 `choices` 数组中的序号，从 `0` 开始。
    </ResponseField>

    <ResponseField name="choices[].message.role" type="string">
      固定值 `"assistant"`，表示此消息由模型生成。
    </ResponseField>

    <ResponseField name="choices[].message.content" type="string">
      模型生成的文本内容。
    </ResponseField>

    <ResponseField name="choices[].finish_reason" type="string">
      生成结束的原因：
      - `"stop"`：模型自然结束生成，或触发了停止序列
      - `"length"`：达到 `max_tokens` 限制被截断
      - `null`：流式模式下尚未完成时的中间状态
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="usage" type="object">
  本次请求的 Token 消耗统计。

  <Expandable title="usage 对象字段">
    <ResponseField name="usage.prompt_tokens" type="integer">
      输入消息（包括系统提示词）消耗的 Token 数。
    </ResponseField>

    <ResponseField name="usage.completion_tokens" type="integer">
      模型生成的回复内容消耗的 Token 数。
    </ResponseField>

    <ResponseField name="usage.total_tokens" type="integer">
      本次请求消耗的 Token 总数，等于 `prompt_tokens + completion_tokens`。
    </ResponseField>
  </Expandable>
</ResponseField>

## 响应示例

```json
{
  "id": "chatcmpl-abc123",
  "object": "chat.completion",
  "created": 1699000000,
  "model": "claude-opus-4-8",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "REST API 设计最佳实践包括：使用名词表示资源、合理使用 HTTP 动词、返回有意义的状态码..."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 25,
    "completion_tokens": 150,
    "total_tokens": 175
  }
}
```

## 流式响应

当请求参数中设置 `"stream": true` 时，API 将以 [Server-Sent Events（SSE）](https://developer.mozilla.org/en-US/docs/Web/API/Server-sent_events) 格式逐步返回生成内容。每个事件以 `data: ` 开头，包含一个 JSON 对象（即 `chunk`）。当生成完成时，服务器将发送 `data: [DONE]` 作为结束标志。

流式响应的 `choices[].delta` 字段替代了非流式响应中的 `choices[].message`，每个 chunk 只包含当前片段的新增内容：

```
data: {"id":"chatcmpl-abc123","object":"chat.completion.chunk","choices":[{"delta":{"content":"REST"},"index":0}]}
data: {"id":"chatcmpl-abc123","object":"chat.completion.chunk","choices":[{"delta":{"content":" API"},"index":0}]}
data: [DONE]
```

<Note>
  流式模式下，`usage` 字段通常仅在最后一个有效 chunk 中返回（部分模型可能不返回）。如需精确统计 Token 消耗，建议使用非流式模式，或在服务端汇总流式响应。
</Note>
