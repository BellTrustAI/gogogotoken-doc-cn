GogoToken API 使用 Bearer Token 进行认证。每个请求都必须在 HTTP 请求头中包含有效的 API 密钥。未携带密钥或密钥无效的请求将被拒绝，并返回 `401` 错误响应。

## 认证头格式

在每个 API 请求的 HTTP 请求头中加入以下字段，将 `YOUR_API_KEY` 替换为你的真实密钥：

```
Authorization: Bearer YOUR_API_KEY
```

以下是使用 cURL 发送带认证头请求的完整示例：

```bash
curl https://gogotoken.org/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "user", "content": "你好！"}
    ]
  }'
```

<Note>
  请妥善保管你的 API 密钥，不要将其硬编码在前端代码或公开的代码仓库中。建议通过环境变量或密钥管理服务进行存储。前往 [API 密钥管理页面](/account/api-keys) 查看或创建你的密钥。
</Note>

## SDK 中的认证

如果你使用 OpenAI 兼容的官方 SDK，只需在初始化客户端时同时指定 `apiKey` 和 `baseURL` 即可完成认证配置。

**Node.js（openai npm 包）**

```javascript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "YOUR_API_KEY",
  baseURL: "https://gogotoken.org/v1",
});

const response = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: "你好！" }],
});

console.log(response.choices[0].message.content);
```

**Python（openai pip 包）**

```python
from openai import OpenAI

client = OpenAI(
    api_key="YOUR_API_KEY",
    base_url="https://gogotoken.org/v1",
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "你好！"}],
)

print(response.choices[0].message.content)
```

## 认证错误响应

当请求未携带 API 密钥、密钥格式错误或密钥已失效时，API 将返回 HTTP `401` 状态码，响应体示例如下：

```json
{
  "error": {
    "type": "authentication_error",
    "message": "Invalid API key provided",
    "code": 401
  }
}
```

收到此错误时，请检查以下几点：

- `Authorization` 请求头是否存在且格式正确（`Bearer ` 前缀不可缺少）
- API 密钥是否有效且未被撤销
- 是否使用了正确的 Base URL `https://gogotoken.org/v1`
