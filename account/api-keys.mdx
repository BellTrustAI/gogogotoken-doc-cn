API 密钥是你调用 GogoToken 服务的凭证。在控制台中创建密钥后，将其用于所有 API 请求的 `Authorization` 头中，即可访问 Claude、GPT 等全部在线模型。

## 创建 API 密钥

<Steps>
  <Step title="登录控制台">
    前往 [https://gogotoken.org](https://gogotoken.org) 并使用你的账号登录。
  </Step>
  <Step title="进入密钥管理页面">
    在控制台左侧导航栏中，点击「API 密钥」或「密钥管理」进入密钥管理页面。
  </Step>
  <Step title="创建新密钥">
    点击「创建新密钥」按钮，可选择为密钥填写一个备注名称（如「开发测试」「生产环境」），便于日后区分和管理。
  </Step>
  <Step title="复制并保存密钥">
    密钥创建成功后，系统将在页面上显示完整的密钥字符串。请立即复制并存放在安全的位置。
  </Step>
</Steps>

<Warning>
  API 密钥创建后**只会显示一次**，页面关闭或刷新后将无法再次查看完整密钥。请立即复制并存放在安全位置（如密码管理器）。如果不慎遗失，需要在控制台删除旧密钥并重新创建一个新密钥。
</Warning>

## 在请求中使用密钥

所有发往 GogoToken API 的请求都需要在 HTTP 请求头中携带密钥进行鉴权：

```
Authorization: Bearer YOUR_API_KEY
```

推荐通过环境变量管理密钥，避免将密钥明文写入代码：

```bash
# macOS/Linux
export GOGOTOKEN_API_KEY="your-api-key-here"

# Windows PowerShell
$env:GOGOTOKEN_API_KEY = "your-api-key-here"
```

在代码中读取环境变量：

```python
import os

api_key = os.environ.get("GOGOTOKEN_API_KEY")
```

详细的鉴权用法请参阅 [鉴权指南](/authentication)。

## 安全建议

妥善保管 API 密钥至关重要——密钥一旦泄露，他人即可以你的名义消耗账户余额。请遵循以下最佳实践：

- **不要将密钥硬编码在代码中**：硬编码的密钥很容易随代码一起泄露，始终使用环境变量或密钥管理服务。
- **不要将密钥提交到 Git 仓库**：在项目根目录的 `.gitignore` 文件中添加 `.env`，防止包含密钥的配置文件被意外提交。
- **不要在前端 JavaScript 中使用密钥**：客户端代码对所有访问页面的用户可见，密钥一旦写入前端代码即等同于公开。所有 API 调用应在后端服务中发起。
- **定期轮换密钥**：养成定期更换密钥的习惯，尤其是在密钥可能已经泄露（如代码曾被公开、团队成员变动）后，应立即删除旧密钥并生成新密钥。

## 删除与轮换密钥

你可以随时在控制台的「API 密钥」页面删除已有密钥。删除操作立即生效，使用该密钥的所有请求将即时被拒绝，不会有任何宽限期。

建议在创建好新密钥并完成替换部署后，再删除旧密钥，以避免服务中断。

如需了解如何在 API 请求中正确使用密钥，请参阅 [鉴权指南](/authentication)。
