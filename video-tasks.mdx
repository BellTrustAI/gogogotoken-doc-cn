视频生成任务提交后，通过 `task_id` 轮询查询任务状态与生成结果。**SD2 推荐路径**：`GET /v1/video/generations/{task_id}`。

<Note>
  两种查询路径返回结构不同，请与提交路径保持一致：
  - **`GET /v1/video/generations/{task_id}`**（SD2 主路径）→ `{code, data: TaskDto}`，`status` 为大写（如 `IN_PROGRESS`、`SUCCESS`），视频地址在 `data.result_url`，上游原始 JSON（含 `usage`）在 `data.data`
  - **`GET /v1/videos/{task_id}`**（OpenAI 兼容）→ 扁平 `{id, object, status, progress, metadata.url}`，`status` 为小写（如 `in_progress`、`completed`）
</Note>

<RequestExample>
```bash SD2 主路径
curl https://gogogotoken.com/v1/video/generations/task_1234567890 \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</RequestExample>

<ResponseExample>
```json 处理中
{
  "code": "success",
  "data": {
    "task_id": "task_1234567890",
    "status": "IN_PROGRESS",
    "progress": "50%"
  }
}
```

```json 生成成功
{
  "code": "success",
  "data": {
    "task_id": "task_1234567890",
    "status": "SUCCESS",
    "progress": "100%",
    "result_url": "https://assets.example.com/generated-video.mp4",
    "data": {
      "status": "succeeded",
      "content": {
        "video_url": "https://assets.example.com/generated-video.mp4"
      },
      "usage": {
        "completion_tokens": 108900,
        "total_tokens": 108900
      }
    }
  }
}
```

```json OpenAI 兼容路径
{
  "id": "task_1234567890",
  "object": "video",
  "model": "doubao-seedance-2-0",
  "status": "completed",
  "progress": 100,
  "metadata": {
    "url": "https://assets.example.com/generated-video.mp4"
  }
}
```
</ResponseExample>

## 路径参数

<ParamField path="task_id" type="string" required>
  视频生成任务 ID（提交响应中的 `id` / `task_id`）。
</ParamField>

## 请求头

<ParamField header="Authorization" type="string" required>
  API Key 鉴权信息，格式为 `Bearer YOUR_API_KEY`。
</ParamField>

## 响应体（`/v1/video/generations/{task_id}`）

<ResponseField name="data.status" type="string">
  任务状态：`QUEUED`、`IN_PROGRESS`、`SUCCESS`、`FAILURE` 等。
</ResponseField>

<ResponseField name="data.progress" type="string">
  进度百分比字符串，如 `"50%"`。
</ResponseField>

<ResponseField name="data.result_url" type="string">
  生成成功后的视频 URL（平台提取自上游响应）。
</ResponseField>

<ResponseField name="data.data.usage.completion_tokens" type="integer">
  Token 计费模型（`doubao-seedance-2-0` / `-fast`）的上游真实输出 token，用于差额结算。
</ResponseField>

## 轮询建议

状态通常按以下顺序变化：

```text
QUEUED / IN_PROGRESS -> SUCCESS / FAILURE
```

成功后的 `result_url` 约 24 小时内有效，请及时 [下载](/video-tools)。
