GogoToken 使用标准的 Bearer Token 方式进行身份认证。每一个发往 GogoToken 的请求都必须在 HTTP 请求头中携带您的 API 密钥，平台以此验证您的身份并关联相应的计费账户。

## 认证方式

所有 API 请求均需在 `Authorization` 请求头中传入以下格式的密钥：

```
Authorization: Bearer YOUR_API_KEY
```

以下是一个使用 cURL 发送认证请求的完整示例：

```bash
curl https://gogotoken.org/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "claude-opus-4-8",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

## 在 SDK 中配置

GogoToken 与 OpenAI SDK 完全兼容。在初始化客户端时，将 `apiKey`（JavaScript）或 `api_key`（Python）参数设置为您的 GogoToken API 密钥，同时将 `baseURL` / `base_url` 指向 GogoToken 端点即可。

<CodeGroup>
  ```javascript JavaScript / Node.js
  import OpenAI from 'openai';

  const client = new OpenAI({
    apiKey: process.env.GOGOTOKEN_API_KEY, // 从环境变量读取，切勿硬编码
    baseURL: 'https://gogotoken.org/v1',
  });
  ```

  ```python Python
  import os
  from openai import OpenAI

  client = OpenAI(
      api_key=os.environ.get("GOGOTOKEN_API_KEY"),  # 从环境变量读取，切勿硬编码
      base_url="https://gogotoken.org/v1",
  )
  ```
</CodeGroup>

<Warning>
  请勿将 API 密钥以明文形式硬编码到源代码中。一旦代码被公开或泄露，您的密钥将面临被滥用的风险，产生的费用由账户持有人承担。
</Warning>

## 安全最佳实践

<Note>
  **不要将 API Key 提交到 Git 仓库。** 即使是私有仓库也存在权限管理风险。建议在 `.gitignore` 中排除所有包含密钥的配置文件（如 `.env`）。
</Note>

<Note>
  **使用环境变量存储密钥。** 在本地开发时使用 `.env` 文件配合 `dotenv` 等工具加载；在生产环境中通过平台的 Secret 管理功能注入环境变量，避免密钥出现在代码库或日志中。
</Note>

<Note>
  **定期轮换密钥。** 建议定期前往 [API 密钥管理页面](/account/api-keys) 吊销旧密钥并创建新密钥，尤其是在团队成员变动或怀疑密钥泄露时应立即轮换。
</Note>

## 认证错误

当请求认证失败时，API 将返回标准的 HTTP 错误状态码。下表列出了常见的认证相关错误及其处理建议：

| 状态码 | 错误类型 | 原因说明 | 处理建议 |
|--------|----------|----------|----------|
| `401 Unauthorized` | 未授权 | API 密钥缺失、格式错误或已失效 | 检查请求头是否正确包含 `Authorization: Bearer YOUR_API_KEY`，并确认密钥未被吊销 |
| `403 Forbidden` | 禁止访问 | 密钥有效，但账户余额不足或访问权限受限 | 前往 [账户余额页面](/account/balance) 充值，或检查账户是否存在访问限制 |

如需创建新密钥或管理现有密钥，请前往 [API 密钥管理页面](/account/api-keys)。
