本站按模型类型采用不同计费方式：**视频按输出 token 据实结算**、**图片按张固定价**、**素材登记按次少量计费**。实际扣费以控制台日志为准。

## Seedance 2.0 视频（Token 计费）

`doubao-seedance-2-0`、`doubao-seedance-2-0-fast`、`doubao-seedance-2-0-mini` 按输出 token 据实计费。任务提交时会进行额度校验，任务完成后按实际 `completion_tokens` 差额结算。

参考：[火山引擎 Seedance 2.0 计费文档](https://www.volcengine.com/docs/82379/1544106?lang=zh#02affcb8)

### doubao-seedance-2-0

旗舰模型，支持 1080p。

| 输出分辨率 | 纯文本 / 文生（无媒体输入） | 含媒体输入 |
| --- | --- | --- |
| 1080p | 51.00 元/百万 token | 31.00 元/百万 token |
| 480p / 720p | 46.00 元/百万 token | 28.00 元/百万 token |

### doubao-seedance-2-0-fast

快版模型，不支持 1080p。

| 输出分辨率 | 纯文本 / 文生（无媒体输入） | 含媒体输入 |
| --- | --- | --- |
| 480p / 720p | 37.00 元/百万 token | 22.00 元/百万 token |

### doubao-seedance-2-0-mini

轻量模型，不支持 1080p。

| 输出分辨率 | 纯文本 / 文生（无媒体输入） | 含媒体输入 |
| --- | --- | --- |
| 480p / 720p | 23.00 元/百万 token | 14.00 元/百万 token |

<Note>
  暂不支持离线推理。带 `-480p` 等后缀的旧别名模型仍按秒计费，见控制台定价。
</Note>

### 示例计算

`doubao-seedance-2-0-mini`，480p 不含视频，生成 108,900 tokens：

```text
参考费用 = 108900 / 1,000,000 × 23.00
        ≈ ¥2.50
```

`doubao-seedance-2-0`，720p 不含视频，生成 108,000 tokens：

```text
参考费用 = 108000 / 1,000,000 × 46.00
        = ¥4.968
```

## Seedream 图片（按张计费）

| 模型 | 参考单价 |
| --- | --- |
| `doubao-seedream-4.0` | ¥0.20 / 张 |
| `doubao-seedream-4.5` | ¥0.25 / 张 |
| `doubao-seedream-5.0-lite` | ¥0.22 / 张 |

请求成功即按 `n`（生成张数）× 单价扣费。详见 [图片生成](/images)。

## 素材管理（按次计费）

| 接口 | 参考单价 |
| --- | --- |
| `CreateAssetGroup` | 免费 |
| `CreateAsset` | ¥0.10 / 次 |

`CreateAsset` 用于将公网图片/视频 URL 登记到素材组，便于在图生视频中引用 `asset://`。详见 [素材管理](/assets)。

## 豆包对话模型（按 Token 计费）

`doubao-seed-2.0-pro` 等对话模型通过 `POST /v1/chat/completions` 调用，按输入与输出 token 计费。`doubao-seed-2.0-pro` 按上下文长度分档计价。详见 [对话模型](/chat-models)。

## 结算方式

- **视频**：任务完成后根据 `completion_tokens`、模型、分辨率与是否含媒体输入计算最终费用。
- **图片**：请求成功即扣费。
- **素材**：`CreateAsset` 成功即扣费。
- **对话**：按每次请求的 prompt / completion token 计费。

<Info>
  不同套餐、分组或活动价格可能影响最终扣费。建议以控制台定价与请求日志作为结算依据。
</Info>

## 查看用量

- 控制台日志：每次请求的扣费详情
- 控制台定价：各模型倍率与参考单价
- 控制台钱包：余额与充值

## 预扣与结算

视频任务提交时会进行额度校验。任务完成后根据实际输出 token 数结算；余额不足时返回 HTTP 402。
